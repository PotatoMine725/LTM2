Phiên này thực hiện 3 việc chính: sửa lỗi đăng nhập nghiêm trọng, cấu hình output fat jar vào thư mục `dist/`, và tạo script đóng gói app kèm Java runtime để chạy trên máy không có JDK.

## 1. Sửa lỗi đăng nhập (Stream Poisoning Bug)

**File:** `src/main/java/server/ClientHandler.java`

### Mô tả lỗi

Sau khi đăng nhập thất bại, client không thể đăng nhập lại dù nhập đúng tài khoản/mật khẩu. Tất cả lần đăng nhập kế tiếp đều trả về thất bại.

### Nguyên nhân

Trong nhánh xử lý đăng nhập thất bại, server ghi thêm `Protocol.HISTORY_END` vào stream:

```java
// Code lỗi (trước khi sửa)
if (ok) {
    username = user;
    sendHistory();  // sendHistory() đã ghi HISTORY_END ở cuối
} else {
    logEvent("Login failed");
    output.writeUTF(Protocol.HISTORY_END);  // <-- Lỗi: viết thừa vào stream
    output.flush();
}
```

Client chỉ đọc đúng 2 UTF string (response code + message) khi xử lý auth. Chuỗi `HISTORY_END` thừa nằm lại trong buffer TCP. Lần đăng nhập tiếp theo, client đọc nhầm `HISTORY_END` thay vì `AUTH_OK`, khiến `sendAuthCommand()` luôn trả về `false`.

### Cách sửa

Xóa dòng ghi `HISTORY_END` trong nhánh thất bại:

```java
// Code sau khi sửa
if (ok) {
    username = user;
    logEvent("Account logged in: " + user);
    sendHistory();  // sendHistory() gửi HISTORY... rồi HISTORY_END
} else {
    logEvent("Login failed");
    // Không ghi gì thêm — client không gọi loadHistory() khi thất bại
}
```

**Nguyên tắc:** Server chỉ ghi đúng những gì client đọc trong mỗi nhánh code. `HISTORY_END` chỉ có nghĩa sau khi đăng nhập thành công.

## 2. Cấu hình fat jar output vào `dist/`

**File:** `pom.xml`

Thêm `<outputDirectory>` vào cấu hình maven-shade-plugin để fat jar được build thẳng vào thư mục `dist/` (thay vì `target/`):

```xml
<configuration>
    <outputDirectory>${project.basedir}/dist</outputDirectory>
    <finalName>ltm2-chat-1.0.0-all</finalName>
    ...
</configuration>
```

Kết quả: `mvn package` tạo ra `dist/ltm2-chat-1.0.0-all.jar` — sẵn sàng cho bước đóng gói tiếp theo.

## 3. Đóng gói app kèm Java Runtime (jpackage)

**File mới:** `package-dist.bat`

### Mục tiêu

Tạo file EXE chạy được trên máy **không cần cài Java** bằng cách nhúng JRE vào trong gói phân phối.

### Công cụ sử dụng

`jpackage` — có sẵn trong JDK 14+ (project dùng Java 17), không cần thêm dependency.

### Cách hoạt động

Script thực hiện 3 bước:

1. **Build fat jar** — `mvn package -DskipTests` → `dist/ltm2-chat-1.0.0-all.jar`
2. **Đóng gói Server** — `jpackage --type app-image` → `dist/LTM2-Server/LTM2-Server.exe`
3. **Đóng gói Client** — `jpackage --type app-image` → `dist/LTM2-Client/LTM2-Client.exe`

Cả hai gói dùng cùng một fat jar nhưng khác `--main-class`:
- Server: `server.ServerApp`
- Client: `client.ClientLauncher`

### Kết quả

```
dist/
├── ltm2-chat-1.0.0-all.jar       ← fat jar (nguồn)
├── LTM2-Server/
│   ├── LTM2-Server.exe           ← chạy được, không cần JDK
│   └── runtime/                  ← JRE nhúng (~60-80 MB)
└── LTM2-Client/
    ├── LTM2-Client.exe           ← chạy được, không cần JDK
    └── runtime/                  ← JRE nhúng (~60-80 MB)
```

### Cách chạy script

```bat
.\package-dist.bat
```

Hoặc trong Claude Code terminal:

```
! .\package-dist.bat
```
